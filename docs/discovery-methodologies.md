# 50 Substack Publication Discovery Methodologies

Target: 1,000+ quality publications for the hex-index personal library.

## Category 1: Substack Platform Features (1-8)

### 1. Substack Category Leaderboards
Browse top publications by category:
- substack.com/top/technology, /top/science, /top/business, /top/politics, /top/culture, /top/finance, /top/food, /top/sports, /top/music, /top/faith, /top/climate, /top/crypto, /top/health

### 2. Publication Recommendations Chains
Each Substack has a /recommendations page. Start with seed publications, spider outward.
- heathercoxrichardson.substack.com/recommendations
- Follow chains 3-4 levels deep

### 3. Substack Bundles
Grouped publications sold together often share quality/topics.
- Search for bundle announcements and member lists

### 4. Substack Notes Mining
Authors promote other authors in Notes. Search for "check out" "recommend" patterns.

### 5. Substack Featured/Staff Picks
Editorially curated lists from Substack itself.
- substack.com/discover
- substack.com/reads

### 6. Substack Search by Topic
Use Substack's internal search for topic keywords.

### 7. Substack's "Similar to" Suggestions
When viewing a publication, Substack suggests similar ones.

### 8. New/Rising Publications
substack.com/discover - filter by new/trending

## Execution Strategy

### Phase 1: High-Yield Automated
- Substack category pages (methodology 1)
- Recommendations chains (methodology 2)
- Google searches (methodologies 9-20)

### Phase 2: Social Mining
- Reddit threads (methodologies 25-26)
- Hacker News links (methodology 27)

### Phase 3: Professional Networks
- Academic searches (methodologies 29-34)
- Author/creator searches (methodologies 35-38)

### Phase 4: Aggregator Scraping
- Newsletter directories (methodologies 39-44)

### Phase 5: Topic Deep Dives
- Specialized searches (methodologies 45-50)

## Rate Limiting Strategy

- 1 request per 2 seconds for any single domain
- Rotate between methodologies to distribute load
- Cache all results to avoid duplicate requests
- Store partial progress for resumability
